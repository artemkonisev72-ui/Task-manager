# Архитектура клиент-серверного взаимодействия в SvelteKit

Связь между сервером (`+page.server.ts`) и клиентом (`+page.svelte`) в приложении на SvelteKit строится на двух основных концепциях: **Load-функции** (для получения данных) и **Form Actions** (для изменения данных).

Ниже разобран пример на основе доски задач (`boards/[id]`).

## 1. Как данные попадают с сервера на клиент (Чтение)

Когда пользователь открывает страницу доски, происходит следующее:

1. **Сервер (`+page.server.ts`)**: SvelteKit запускает функцию `load()`.
2. Функция обращается к базе данных через Prisma, достает нужную доску (`board`) вместе со всеми ее колонками и задачами, а также список пользователей (`users`).
3. Функция возвращает объект: `return { board, users };`
4. **Клиент (`+page.svelte`)**: Эти данные автоматически передаются в Svelte-компонент через специальное свойство (prop):
   ```svelte
   <script lang="ts">
     // Переменная data содержит то, что вернула функция load на сервере
     export let data; 
     
     // Теперь мы можем использовать data.board и data.users в шаблоне HTML
   </script>
   ```

**Особенности работы под капотом:**
* Если это **первая загрузка страницы**, сервер собирает из этих данных готовый HTML (Server-Side Rendering) и отдает его браузеру.
* Если пользователь **перешел на страницу внутри приложения** (например, кликнул ссылку на доску с главной страницы), SvelteKit просто делает `fetch`-запрос к специальному файлу `/__data.json` на сервере, получает только чистые данные и реактивно обновляет страницу без перезагрузки браузера.

---

## 2. Как клиент отправляет данные на сервер (Запись/Мутация)

Когда пользователю нужно создать, переместить или удалить задачу, он взаимодействует с **Form Actions** (действиями формы).

1. **Клиент (`+page.svelte`)**: В HTML-коде компонента используются стандартные теги `<form>`. В SvelteKit они обычно усиливаются специальной директивой `use:enhance`.
   ```svelte
   <!-- Пример: форма для создания задачи. Action ?/createTask вызывает функцию createTask на сервере -->
   <form method="POST" action="?/createTask" use:enhance>
       <input type="hidden" name="columnId" value={column.id} />
       <input type="text" name="title" placeholder="Имя задачи" />
       <button type="submit">Создать</button>
   </form>
   ```
2. При нажатии на кнопку "Создать", браузер собирает данные из всех `input` внутри этой формы (создает объект `FormData`) и отправляет HTTP `POST`-запрос на текущий URL: `POST /boards/1?/createTask`.
3. Директива `use:enhance` перехватывает этот запрос и отправляет его через JavaScript (AJAX) в фоне, чтобы страница не перезагружалась целиком (типичное поведение HTML-форм).
4. **Сервер (`+page.server.ts`)**: Запрос попадает в объект `actions` и находит функцию `createTask`.
   ```typescript
   export const actions = {
       createTask: async ({ request, locals }) => {
           // Получает данные формы из клиентского POST-запроса
           const data = await request.formData();
           const title = data.get('title');
           
           // Сохраняет новую задачу в базу данных
           await prisma.task.create({ data: { title, ... } });
       }
   }
   ```
5. **Автоматическое обновление UI**: После успешного выполнения action на сервере, SvelteKit автоматически говорит фронтенду: *"Данные изменились"*. Клиент сам заново вызывает `load`-функцию в фоне, получает свежие списки колонок и задач из БД, и интерфейс мгновенно перерисовывается в браузере (задача появляется в колонке).

## Почему этот подход удобен:
* **Тесная интеграция (Типизация)**: Разработчику не нужно писать вручную логику `fetch` к эндпоинтам, обрабатывать ответы, парсить JSON и обновлять локальные стейты. SvelteKit берет всю эту рутину на себя.
* **Безопасность**: Весь код внутри `+page.server.ts` выполняется **строго на сервере**. Можно смело обращаться к базе данных, использовать секретные ключи или проверять права доступа — этот код никогда не попадет в браузер клиента.
* **Работа без JS**: Если у пользователя выключен JavaScript, формы всё равно будут работать (конечно, с полной перезагрузкой страницы браузером), так как они используют стандартный механизм браузера `method="POST"`. (С `enhance` они просто работают быстрее и плавнее).
